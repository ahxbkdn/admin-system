# 课程管理功能测试计划

## 1. 测试目标
验证课程管理系统的完整功能，包括添加课程、添加章节、修改章节、删除章节等操作，确保所有功能都能正确调用后端接口并执行。

## 2. 测试环境
- 前端：React + Ant Design
- 后端：Flask + Flasgger
- 服务地址：http://localhost:8084
- 前端地址：http://localhost:5173

## 3. 测试步骤

### [x] 任务1：验证后端服务状态
- **优先级**：P0
- **依赖**：无
- **描述**：检查后端服务是否正常运行，API接口是否可访问
- **成功标准**：后端服务运行在8084端口，Swagger文档可访问
- **测试要求**：
  - `programmatic` TR-1.1: 访问 http://localhost:8084/swagger-ui/ 应返回200状态码
  - `programmatic` TR-1.2: 访问 http://localhost:8084/api/courses 应返回课程列表

### [x] 任务2：测试添加课程功能
- **优先级**：P0
- **依赖**：任务1
- **描述**：测试通过前端界面添加课程的功能
- **成功标准**：课程能够成功添加到系统中
- **测试要求**：
  - `programmatic` TR-2.1: 调用 POST /api/courses 接口应返回200状态码
  - `human-judgement` TR-2.2: 前端界面应显示添加成功提示
  - `programmatic` TR-2.3: 课程应出现在课程列表中

### [x] 任务3：测试添加章节功能
- **优先级**：P0
